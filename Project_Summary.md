     - **정면 영역 ($0^\circ \sim 45^\circ$)**: 맑게 튕기는 3음 유리 벨 (1800Hz ➔ 2400Hz ➔ 2700Hz) + 볼륨 100% ➔ **화사한 "또로롱-! 🎶"**
     - **측면 영역 ($45^\circ \sim 135^\circ$)**: 청량한 미니 톡- 톤 (1500Hz) ➔ **"톡-"**
     - **후방 영역 ($135^\circ \sim 180^\circ$)**: 어두운 저음 톤 (500Hz ➔ 350Hz) + 볼륨 10% 감쇄 ➔ **어두운 "뾴..."**

   - **거리 기반 펄스 주기(BPM) 조절**: 30m 이상(2.0초 간격) ➔ 10~30m(1.2초 간격) ➔ 3~10m(0.6초 간격) ➔ 3m 이내 도착 차임벨 재생 및 자동 종료.

## 1. 프로젝트 개요
- **목적**: 사용자의 음성 및 위치(GPS/Heading)를 실시간으로 인식하여, 미리 준비된 관광지 데이터를 바탕으로 정확한 안내 및 3D 공간 음향 내비게이션을 제공하는 지능형 웹 애플리케이션.
- **기술 스택**: HTML, CSS, Vanilla JavaScript, Web Audio API, DeviceOrientation API, Vite
- **배포**: GitHub Pages + GitHub Actions (CI/CD 자동화)

## 2. 주요 구현 기능 및 특징

### 2.1. 사용자 인터페이스 (UI/UX)
- **Glassmorphism 기반 다크 모드**: 세련된 반투명 유리 질감(`.glass-panel`)과 심플한 다크 테마 적용.
- **모바일 최적화 (반응형)**: 모바일 환경에서 텍스트가 길어졌을 때 마이크 버튼을 가리지 않도록 웹페이지 전체 단위의 스크롤 및 하단 여백(`padding-bottom: 4rem`) 적용.
- **실시간 GPS 디버깅 박스**: 보정 좌표, 원본 좌표(Raw), 수신 오차 범위(Accuracy) 실시간 표시.

### 2.2. 오디오 피드백 및 3D 공간 음향 (Web Audio API)
외부 음원 파일 없이 자바스크립트의 `AudioContext`를 활용하여 상황에 맞는 효과음 및 3D 오디오 가이드를 직접 합성(Synthesis)하여 제공.
- **마이크 켜짐/꺼짐음**: 삐빅 하는 명료한 시스템음.
- **로딩 대기음**: Gemini의 답변을 기다리는 동안 맑은 방울 소리 재생.
- **3D 공간 음향 비콘**: 목적지 방위각 및 나침반 Heading에 따라 좌/우 3D Panning과 거리별 펄스 주기를 조절하는 보행자용 내비게이션 사운드.

### 2.3. 클라이언트 사이드 RAG (검색 증강 생성)
백엔드나 DB 없이 프론트엔드 환경에서 자체적인 RAG 시스템을 구축하여 환각(Hallucination) 현상 방지.
- **데이터 분리**: 유지보수를 위해 관광지 데이터를 `tourData.js`로 모듈화.
- **고도화된 검색 로직**: 정규식 및 괄호 부연설명 제외 정제로 사용자의 질문에서 유적지/건물명 추출.

### 2.4. 지능형 TTS 및 포맷팅 (Text-to-Speech)
- **기기별 재생 속도 최적화**: PC(1.8배속) 및 모바일(1.2배속) 환경 자동 구분.
- **TTS 낭독용 텍스트 정제**: 특수문자(`*`, `#`) 제거 및 `getJosa` 함수로 받침 조사를 완벽하게 자동 선택.

## 3. 실행 및 관리 방법
- **로컬 개발 모드**: `npm run dev`
- **배포 및 업데이트**: 코드를 수정한 뒤 GitHub의 `main` 브랜치에 커밋 및 푸시(Push)하면, `.github/workflows/deploy.yml` 설정에 의해 자동으로 빌드되어 `https://kjwtuat.github.io/tinytour/` 주소로 배포됨.
