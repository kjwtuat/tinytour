# tinytour (모두의 투어) 프로젝트 요약 정리

## [2026-07-22] 최근 업데이트 내역
- **TTS 자연어 처리 고도화 (`main.js`)**: 가장 가까운 위치 안내 시 한국어 조사가 어색한 문제(예: "일청담가 가장 가깝습니다")를 해결하기 위해 `getJosa` 함수를 도입하여 종성(받침) 유무에 따라 '이/가', '을/를', '은/는'을 완벽하게 자동 선택하도록 개선.
- **데이터베이스 갱신 (`tourData.js`)**: 새롭게 개발된 `tinymap` 에디터를 통해 추출된 캠퍼스 도보망 좌표 데이터 23개로 기존 데이터를 대체(업데이트) 완료.

---

본 문서는 `TourAgent` 웹 기반 AI 음성 관광 비서의 전체 개발 내용과 주요 기능, 그리고 기술적 특징을 정리한 문서입니다.

## 1. 프로젝트 개요
- **목적**: 사용자의 음성을 실시간으로 인식하여, 미리 준비된 관광지 데이터를 바탕으로 정확하게 답변을 제공하고 이를 다시 음성(TTS)으로 읽어주는 지능형 웹 애플리케이션.
- **기술 스택**: HTML, CSS, Vanilla JavaScript, Vite
- **배포**: GitHub Pages + GitHub Actions (CI/CD 자동화)

## 2. 주요 구현 기능 및 특징

### 2.1. 사용자 인터페이스 (UI/UX)
- **Glassmorphism 기반 다크 모드**: 세련된 반투명 유리 질감(`.glass-panel`)과 심플한 다크 테마 적용.
- **모바일 최적화 (반응형)**: 모바일 환경에서 텍스트가 길어졌을 때 마이크 버튼을 가리지 않도록 웹페이지 전체 단위의 스크롤 및 하단 여백(`padding-bottom: 4rem`) 적용.
- **상태별 애니메이션**:
  - 마이크 대기 중: 기본 상태
  - 음성 인식 중: 마이크 버튼이 붉은색으로 변하며 파동(Pulse) 애니메이션 재생.
  - 답변 대기 중: 마이크 아이콘이 사라지고 테두리가 빙글빙글 도는 로딩(Spin) 스피너로 전환.

### 2.2. 오디오 피드백 (Web Audio API)
외부 음원 파일 없이 자바스크립트의 `AudioContext`를 활용하여 상황에 맞는 효과음을 직접 합성(Synthesis)하여 제공.
- **마이크 켜짐/꺼짐음**: 삐빅 하는 명료한 시스템음.
- **로딩 대기음**: Gemini의 답변을 기다리는 동안 1초 간격으로 맑고 경쾌한 방울 소리(1200Hz ➔ 1500Hz)가 재생되어 청각적 피드백 제공.

### 2.3. 클라이언트 사이드 RAG (검색 증강 생성)
백엔드나 DB 없이 프론트엔드 환경에서 자체적인 RAG 시스템을 구축하여 환각(Hallucination) 현상 방지.
- **데이터 분리**: 유지보수를 위해 관광지 데이터 35여 종을 `tourData.js`로 모듈화.
- **고도화된 검색 로직**: 
  - 복합 명사(예: "청운교 백운교")와 괄호 부연설명(예: "석가탑 (삼층석탑)")을 정규식을 통해 개별 단어로 쪼개어 검색망(Keywords) 형성.
  - 사용자의 질문("청운교와 백운교 어때?", "목어에 대해 알려줘") 속에서 정확한 타겟 데이터를 끄집어내는 유연한 매칭 지원.
- **프롬프트 주입**: 매칭된 데이터가 있을 경우, 해당 유적지의 한/영 공식 소개글을 프롬프트 최상단에 주입하여 공식 데이터 기반의 짧은 요약 답변 유도.

### 2.4. 지능형 TTS 및 포맷팅 (Text-to-Speech)
- **기기별 재생 속도 최적화**: PC와 스마트폰 브라우저의 TTS 엔진(속도 해석)이 다른 점을 고려하여, `User-Agent` 기반으로 기기를 감지.
  - PC 환경: 1.8배속 적용
  - 스마트폰(Mobile) 환경: 1.2배속 적용 (과속 방지)
- **TTS 낭독용 텍스트 정제**: 
  - Gemini에게 특수문자, 마크다운, 이모지 사용을 엄격히 금지하는 프롬프트 적용.
  - 출력된 텍스트에서 정규식(`/[*\#]/g`)을 통해 불필요한 기호를 한 번 더 강제 삭제하여 TTS가 "별표"라고 읽는 등 어색한 상황을 원천 차단.

## 3. 실행 및 관리 방법
- **로컬 개발 모드**: `npm run dev` (실시간 변경 사항 확인 가능)
- **배포 및 업데이트**: 코드를 수정한 뒤 GitHub의 `main` 브랜치에 커밋 및 푸시(Push)하면, `.github/workflows/deploy.yml` 설정에 의해 자동으로 빌드되어 `https://kjwtuat.github.io/sullivan_tour/` 주소로 배포됨.
